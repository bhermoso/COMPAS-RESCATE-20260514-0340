## PAN-001

Panóptico textual provisional — PROVISIONAL_NO_VALIDADO

**Propósito del bloque.** Las líneas 1 a 2000 de COMPÁS contienen los estilos CSS del bloque `<style>` inicial. En el rango auditado se han identificado 30 entidades visuales con evidencia directa. No puede afirmarse que constituyen la totalidad de la capa visual del sistema sin auditar el rango completo (l.2001+). No existe lógica funcional en este rango salvo un único listener de error en l.27–41.

**Composición.** Un único archivo HTML con un bloque `<style>` de aproximadamente 1960 líneas CSS activas que cubre 30 entidades visuales (más 1 transversal responsive) distribuidas en 9 capas funcionales. El bloque no cierra en l.2000 — continúa en CAP-002.

**Sistema de control de estado.** 20 contratos visuales (CON-001 a CON-020) definen pares de clases CSS (base/modificador) observados en el rango. El patrón dominante es `clase_base { display:none } clase_base.modificador { display:block }`. Si JavaScript aplica la clase modificadora, el efecto CSS se produce; el productor JS no ha sido observado en el rango. Las clases identificadas son: .visible (CON-001), .activa (CON-002, CON-007, CON-012), .abierto/.abierta (CON-003, CON-008, CON-009, CON-014), .activo (CON-004, CON-005, CON-006, CON-010, CON-011, CON-013), .visible (CON-017), .drag-over (CON-018), .closing (CON-019), @media print (CON-020).

**Dependencias externas.** 6 CDN en l.13–25. Firebase es la única con consumidor JS observado en el rango (listener l.27-41). Mammoth.js cuenta con un comentario CSS (l.808) que declara el propósito del wrapper `.compas-informe-html`; la invocación JS no ha sido leída en el rango (REL-001 CSS_PARCIAL). Las otras 4 dependencias tienen evidencia probable o pendiente de validación. QRCodeJS es la única sin relación CSS observable.

**Riesgo sistémico mayor.** El color de fondo de al menos 5 componentes (.propuesta-linea-header, .modal-detalle-header, .agenda-tipo-linea-codigo, .badge-linea, .doc-linea) depende de JavaScript aplicando inline styles. Si JS no funciona o el dato de color está ausente, estos componentes muestran texto blanco sobre fondo blanco — invisible para el usuario. No detectable desde CSS solo. Nota: .modal-header tiene background CSS definido (linear-gradient #94d40b, l.1618) y no depende de JS.

**Deuda cromática.** 11 deudas documentadas. La más sistémica: `linear-gradient(X, color 0%, mismo_color 100%)` — degradado de un único color sin efecto visual — aparece al menos 6 veces (DV-007). La más impactante: uso de paleta Tailwind ajena al sistema (#ea580c, #f97316, #1e40af, #b91c1c, #047857) en componentes de alta visibilidad.

**Sistema de notificación.** ENT-029 (toast, z-index:10000) es el componente de feedback con mayor z-index identificado en el rango l.1–2000. No puede afirmarse desde el rango auditado que toda operación asíncrona del sistema depende de él. No existe overlay de carga a pantalla completa en este bloque.

**Complejidad de media queries.** 5 bloques @media dispersos en MR-029, 036, 042, 043, 044 para el breakpoint 768px. Sin punto de definición centralizado. El breakpoint 600px aparece en MR-040 y MR-044. El breakpoint 1200px aparece solo en MR-042.

**Jerarquía de capas z-index.** Registrada en CSS en 3 niveles: z-index:100 (menú contextual tarjeta Kanban, l.1482) < z-index:1000 (overlay modal, l.1614) < z-index:10000 (toast, l.1794). Los valores CSS declarados no presentan solapamiento entre sí. La ausencia de conflictos en DOM real requiere verificación de runtime fuera del rango auditado.
